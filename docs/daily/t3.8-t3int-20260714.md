## T3.8 — OTA en Android físico (RF-23) ✅

### Flujo ejecutado

| Paso | Estado | Evidencia |
|---|---|---|
| Instalar v1 (`version_code=1`) | ✅ | `adb install` tras activar **Instalar vía USB** (MIUI) |
| Arrancar app → detectar v100 | ✅ | Diálogo **Actualización disponible** · Versión 1.0.100 · release notes T3.8 |
| Descargar APK | ✅ | `GET /app-release.apk` en servidor `:8765` (vía `adb reverse`) |
| Instalar v100 | ✅ | Diálogo sistema **Peligro** MIUI → aceptar → `versionCode=100` |

### Comandos clave

```bash
adb reverse tcp:8080 tcp:8080   # API local
adb reverse tcp:8765 tcp:8765   # APK download
# APK v1 → API http://127.0.0.1:8080
# POST /app/register-version → version_code=100, apk_url=http://127.0.0.1:8765/app-release.apk
```

### Verificación final

```
adb shell dumpsys package com.sentilife.app | grep versionCode
→ versionCode=100  versionName=1.0.100
```

**Nota técnica:** WiFi con aislamiento AP impedía descarga LAN directa (`192.168.1.38:8765` → 0%). Solución: `adb reverse` por USB.
